## Abstract

Speaker confidence influences how speech is perceived in education, interviews, and public communication, yet it is usually assessed subjectively. This project implements an end-to-end system that estimates reading confidence from speech audio alone, without wearable sensors or external physiological devices.

The completed system consists of three integrated stages. First, frame-level breathing activity is detected using a deep neural network (**ProposedNet**) that combines a Conformer encoder with a bidirectional LSTM. The model was trained on 8,000 read-speech utterances with frame labels derived from a rule-based breath-detection pipeline applied to LibriSpeech-derived recordings. Second, breathing patterns and complementary acoustic features are aggregated into a 21-dimensional feature vector used to classify confidence into three levels: **Low**, **Medium**, and **High**. Sixteen machine-learning methods were compared; a **Stacking Ensemble** achieved the best cross-validated performance and was selected for deployment. Third, a Flask web application provides a browser-based interface for uploading or recording audio and displaying breath statistics, probability curves, and confidence predictions in real time.

Training of the breath-detection model was performed on Google Colab using a **Tesla T4 GPU** (15.6 GB). A full 43-epoch run required approximately **29 minutes**, with each epoch taking about 40 seconds. Inference and the deployed web application run on **CPU**, where a typical 20–30 second clip is processed in a few seconds.

On the held-out breath-validation set, the best checkpoint (epoch 39) reached **97.7% frame accuracy**, **61.7% precision**, **75.8% recall**, and **68.0% F1-score**. On the independent confidence test set (120 utterances), the Stacking Ensemble reached **70.0% accuracy** and **70.6% macro F1**, outperforming the rule-based formula baseline (40.0% test accuracy).

**Keywords:** speaker confidence, breath detection, Conformer, BiLSTM, speech processing, ensemble learning, Flask deployment

# 1. Introduction

### 1.1 Background

Speech carries more information than the words themselves. Pauses, breathing rhythm, energy variation, and spectral structure all reflect how a speaker controls delivery. In read-aloud and presentation tasks, these cues correlate with perceived confidence: steady breathing and regular pacing often accompany controlled speech, whereas irregular pauses and unstable breath patterns may suggest hesitation or reduced confidence.

Recent work in paralinguistic speech processing has shown that physiological information—especially breathing—can be inferred from audio without contact sensors. Deep architectures such as the Conformer, originally developed for speech recognition, have also proven effective for frame-level event detection in speech, including disfluency and breath localization.

Our semester-1 report (*ENCS5200-Section2-Report*, February 2026) outlined a three-module pipeline: rule-based breath detection, neural breath modeling, and confidence inference potentially involving heart-rate-related cues. During the second semester, we implemented, trained, and deployed the system described in this document. Several design choices evolved during implementation; Section 3 summarizes what changed and why.

### 1.2 Motivation

Manual confidence assessment is slow, subjective, and difficult to scale. Automated tools that operate on speech alone could support language learners, presentation training, and standardized communication evaluation. Because breathing is tightly coupled with speech production, it offers a practical and non-invasive signal for this purpose.

### 1.3 Problem Statement

The project addresses the following question: **Can reading confidence be estimated automatically from speech by detecting in-speech breathing events and analyzing the resulting temporal patterns together with standard acoustic features?**

Constraints imposed on the final system:

- Input: mono speech audio only (WAV/MP3/M4A/FLAC/OGG).
- Output: discrete confidence label (Low / Medium / High) plus interpretable breath statistics.
- No external sensors, ECG, or heart-rate estimation pipeline in the deployed system.

### 1.4 Objectives

The objectives achieved in the final implementation are:

1. Build a rule-based pipeline to generate frame-level breath labels for supervised training.
2. Train and validate a Conformer–BiLSTM breath detector on a large read-speech subset.
3. Extract breathing and acoustic features from new audio and classify confidence using a data-driven model.
4. Compare multiple classifiers and select the best-performing method for deployment.
5. Deliver a working web application that integrates both models for end-to-end inference.

### 1.5 Scope Note (Implementation vs. Proposal)

The semester-1 proposal discussed heart-rate estimation from speech-derived breathing via ICA and peak detection. **This component was not implemented in the final system.** After reviewing dataset availability and project timeline, we narrowed the scope to **breathing-pattern analysis plus acoustic features** for confidence classification. Heart-rate-related literature remains relevant background (Section 2) but is not part of the deployed pipeline. Table 1 documents other differences.

# 2. Literature Review

*(Condensed from semester-1 background; expanded with implementation-relevant work.)*

Speech-based physiological inference has been studied for stress, emotion, and cardiac activity. Qian et al. compared knowledge-based openSMILE features with self-supervised Hybrid BYOL-S embeddings for predicting BPM and HRV from the Ulm-TSST corpus, showing strong speaker-dependent performance but limited speaker-independent generalization [1].

Madan et al. proposed estimating confidence from speech by combining speech-derived breathing with heart-rate proxies, reporting that confident speakers tended to show lower estimated heart rates [2]. Their multi-stage pipeline (SBreathNet → ICA → HR estimation) influenced our initial proposal. Our final system adopts the **breath-detection** idea but replaces the HR stage with **direct confidence classification** from breathing statistics and acoustic descriptors.

Schmitt et al. addressed frame-wise breath detection with self-training approaches on conversational speech [3]. Their use of mel-spectrogram, ZCR, and variance-based descriptors aligns with our feature design.

For irregular temporal patterns in speech, Qian et al. demonstrated that a **Conformer + BiLSTM** architecture improves F1 on stuttering detection [4]. We adopted a similar hybrid design—Conformer blocks for local and global modeling, BiLSTM for bidirectional temporal context—applied here to breath/non-breath frame classification.

Steidl et al. predicted children's reading confidence from acoustic–prosodic features using Random Forests [5], supporting our Module III strategy of classical ML on hand-crafted features rather than end-to-end confidence regression.

## 4.2 Module I: Rule-Based Label Generation

Although the semester-1 report described Module I as a full detection pipeline (forced alignment, pause filtering, ZCR/VMS scoring, post-processing), in practice its primary role in the **completed project** was to produce **supervisory frame labels** for Module II training.

The rule-based stage assigns binary labels to 10 ms frames using weighted combinations of normalized ZCR, VMS, and related descriptors, followed by temporal merging and minimum-duration filtering. These labels were precomputed and stored in `labels_train_clean_100.csv`; the neural model then learns to generalize beyond fixed rules by training on the **full waveform**, not only pause regions.

This design choice improved robustness: the Conformer–BiLSTM model sees speech, silence, and breath in context, rather than relying on transcript-aligned pauses at inference time.

## 4.3 Module II: Neural Breath Detection (ProposedNet)

### 4.3.1 Feature Extraction

For each frame:

| Parameter | Value |
|-----------|-------|
| Sample rate | 16,000 Hz |
| Frame length | 400 samples (25 ms) |
| Hop size | 160 samples (10 ms) |
| Mel bands | 64 |
| Power | 2.0 |
| Mel scaling | log-power (`power_to_db`, ref = 1.0) |
| ZCR | librosa zero-crossing rate, transposed to (T, 1) |
| VMS | variance of mel bands across frequency, per frame |
| **Total dimension** | **66** (64 + 1 + 1) |
| Normalization | Per-utterance z-score (mean/std + 1e−8) |

### 4.3.2 Network Architecture

**Figure 2 — ProposedNet architecture**

```
Input (B, T, 66)
    │
    ▼
Conv2D subsample ×2 (1→32→32, stride 2) + ReLU
    │
    ▼
Linear projection (544 → 128) + Dropout(0.2)
    │
    ▼
3 × ConformerBlock (dim=128, heads=4)
    │   each block: FFN → MHA → ConvModule(k=31) → FFN
    ▼
ConvTranspose1d upsample ×2 (restore temporal resolution)
    │
    ▼
BiLSTM (1 layer, hidden 64 × 2 directions → 128)
    │
    ▼
LayerNorm + Linear(128→1) → logits → sigmoid → P(breath)
```

## 4.4 Module III: Confidence Classification

### 4.4.1 Feature Vector (21 dimensions)

**Breathing features (8)** — computed from frame probabilities after threshold **T = 0.65**:

1. Breathing rate (breaths/min)  
2. Mean breath segment duration (s)  
3. Std of breath segment durations  
4. Pause regularity (std of segment start times)  
5. Breath ratio (fraction of breath frames)  
6. Mean breath probability  
7. Max breath probability  
8. Number of breath segments  

**Acoustic features (13):** RMS mean/std/max, ZCR mean/std, spectral centroid mean/std, spectral bandwidth mean, spectral rolloff mean, MFCC-1/2/3 means, speech ratio.

Dataset-wide averages (597 files): breathing rate ≈ **22.45** breaths/min, breath ratio ≈ **0.157**, mean segments ≈ **10.83**.

Most informative features (Random Forest + Extra Trees average importance): `num_segments` (0.144), `spec_bw_mean` (0.069), `spec_centroid_std` (0.061).

### 4.4.2 Classifier Comparison

Sixteen methods were evaluated with **5-fold stratified cross-validation** on SMOTE-augmented train+val data, using a pipeline of RobustScaler → classifier.

**Selection rule:** maximize (Accuracy + F1_macro) / 2 subject to both ≥ 0.70.

**Formula baseline (from semester-1 report):**  
score = 0.35·rate_score + 0.45·exp(−regularity) + 0.20·ratio_score  
Ideal rate: 12–20 breaths/min; ideal breath ratio: 0.05–0.30.

### 4.4.3 Selected Model: Stacking Ensemble

**Base learners:** SVM (RBF, C=10), ExtraTrees (200 est., depth 8), HistGradientBoosting (200 iter), QDA (reg=0.1), GaussianNB  
**Meta-learner:** LogisticRegression (C=1.0)  
**Stacking CV folds:** 5  

Saved artifact: `module3_best_model.pkl`

### Effect of Hardware on Training Time

On the T4 GPU, GPU-accelerated tensor operations (convolution, attention, LSTM) kept each epoch under one minute. Re-running the same configuration on CPU (tested briefly during local integration) increased per-batch latency substantially; a full 43-epoch run would likely require **several hours** on CPU alone, making GPU access important for iterative hyperparameter tuning.

For **inference**, CPU execution is acceptable: the deployed application processes typical classroom-length recordings interactively. GPU would reduce latency further but is not required for the demo environment.

# 7. Discussion

### 7.1 What Worked

1. **Neural breath detection** clearly outperforms using rule-based decisions at inference time; the model generalizes to full audio without transcript alignment.  
2. **Conformer + BiLSTM** handled frame-level temporal context effectively; 3 Conformer blocks provided sufficient capacity without the overfitting observed when larger counts were considered early in experimentation.  
3. **Stacking Ensemble** exploited complementary biases among diverse base learners, yielding the best CV and test performance.  
4. **Breath + acoustic features** capture both respiratory behavior and delivery quality (energy, spectral shape, MFCCs).

### 7.2 Challenges Encountered

1. **Class imbalance (breath frames):** required `pos_weight = 6.0` in BCE loss.  
2. **Label noise from rule-based targets:** limits precision ceiling; manual breath annotations would likely improve Module II.  
3. **Architecture mismatch during deployment:** initial `app.py` used a different network definition than the saved checkpoint; resolving this required aligning code with the training notebook (`ConformerBiLSTM` class).  
4. **sklearn version drift:** models pickled with scikit-learn 1.6.1 emit warnings when loaded on 1.7.2; predictions still run but reproducibility should pin library versions in production.  
5. **Scope reduction:** heart-rate estimation was deferred; the final thesis reports an honest, working subset of the original vision.

### 7.3 Limitations

- Confidence labels depend on human ratings in ReadingConfidenceDataset; inter-rater variability propagates to model evaluation.  
- Three-class output does not provide a continuous confidence score as originally proposed.  
- System trained primarily on read English speech; generalization to spontaneous speech or other languages is untested.  
- Real-time factor on CPU is acceptable for offline analysis but not optimized for streaming.

# 8. Conclusion and Future Work

We designed, implemented, trained, and deployed a speech-based system for reading confidence assessment built around in-speech breath detection. Module II (ProposedNet) achieves **68.0% F1** on frame-level breath validation. Module III (Stacking Ensemble) achieves **70.0% test accuracy** and **70.6% macro F1** on three-class confidence classification, a substantial improvement over the rule-based formula baseline.

The Flask web application demonstrates end-to-end functionality: users can upload or record speech and receive interpretable breath analytics together with a confidence label.

**Future work:**

1. Integrate heart-rate estimation or other physiological proxies as originally proposed, if synchronized data become available.  
2. Collect manual breath annotations on a subset of LibriSpeech data to refine Module II labels.  
3. Explore end-to-end confidence prediction directly from neural embeddings.  
4. Pin dependency versions and containerize deployment (Docker) for reproducibility.  
5. Evaluate on spontaneous speech and Arabic read-aloud data.  
6. Export continuous confidence scores via calibrated class probabilities.
